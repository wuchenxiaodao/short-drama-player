# 短剧互动播放器全栈重构设计文档

## 概述

对短剧互动播放器项目进行渐进式全栈重构，以测试覆盖为优先，逐步提升代码质量、性能和用户体验。

## 重构目标

1. **测试覆盖优先** - 建立完整的测试基础
2. **代码质量** - 改善代码结构、命名、分层
3. **性能优化** - 优化加载速度、响应时间
4. **UI/UX 改进** - 提升用户体验和视觉效果

## 重构顺序

### 第一阶段：建立测试基础（1-2天）

#### 1.1 后端单元测试

**目标**: Service 层测试覆盖率 80%+

**任务**:
- 添加 JUnit 5 + Mockito 依赖
- 为每个 Service 类创建单元测试
- 使用 H2 内存数据库进行测试隔离
- 创建测试数据工厂类

**测试类规划**:
```
backend/src/test/java/com/drama/
├── service/
│   ├── DramaServiceTest.java
│   ├── EpisodeServiceTest.java
│   ├── InteractionServiceTest.java
│   ├── ProgressServiceTest.java
│   ├── AuthServiceTest.java
│   ├── CommentServiceTest.java
│   ├── RatingServiceTest.java
│   └── UserProfileServiceTest.java
├── controller/
│   ├── DramaControllerIntegrationTest.java
│   ├── EpisodeControllerIntegrationTest.java
│   └── ...
└── factory/
    └── TestDataFactory.java
```

#### 1.2 后端集成测试

**目标**: Controller 层测试覆盖率 70%+

**任务**:
- 使用 @SpringBootTest 进行集成测试
- 测试 API 端点的请求/响应
- 测试异常处理和边界情况

### 第二阶段：后端代码重构（2-3天）

#### 2.1 Service 层优化

**问题**:
- 部分 Service 方法过长
- 职责不够单一
- 缺少事务管理

**改进**:
- 拆分过长方法为多个小方法
- 明确每个方法的职责
- 添加 @Transactional 注解
- 优化数据库查询

#### 2.2 Controller 层优化

**问题**:
- 响应格式不统一
- 参数校验分散
- 异常处理不一致

**改进**:
- 统一响应格式（创建 ApiResponse 类）
- 使用 @Valid 进行参数校验
- 创建全局异常处理器

#### 2.3 Repository 层优化

**问题**:
- 部分查询效率低
- 缺少自定义查询

**改进**:
- 优化 N+1 查询问题
- 添加自定义查询方法
- 使用 JPQL 提高查询效率

### 第三阶段：前端代码重构（2-3天）

#### 3.1 代码模块化

**当前问题**:
- preview.html 文件过大（1343行）
- 代码混杂（HTML/CSS/JS）
- 难以维护和测试

**重构方案**:
```
frontend/
├── index.html          # 主页面结构
├── css/
│   ├── styles.css      # 主样式
│   ├── player.css      # 播放器样式
│   └── interaction.css # 互动样式
├── js/
│   ├── app.js          # 主应用逻辑
│   ├── player.js       # 播放器相关
│   ├── interaction.js  # 互动系统
│   ├── api.js          # API 调用封装
│   ├── utils.js        # 工具函数
│   └── state.js        # 状态管理
└── assets/
    └── images/
```

#### 3.2 性能优化

**任务**:
- 优化图片加载（懒加载）
- 减少 DOM 操作
- 优化动画性能（使用 CSS 动画）
- 减少重绘和回流

#### 3.3 代码质量

**任务**:
- 使用 ESLint 进行代码检查
- 添加 JSDoc 注释
- 统一代码风格
- 创建可复用组件

### 第四阶段：UI/UX 改进（1-2天）

#### 4.1 交互动画优化

**任务**:
- 优化投票动画（表情雨）
- 改进答题界面反馈
- 增强抉择系统视觉效果
- 优化彩蛋触发动画

#### 4.2 响应式设计

**任务**:
- 优化移动端适配
- 改进触摸交互
- 优化横屏模式

#### 4.3 用户体验细节

**任务**:
- 添加加载状态提示
- 优化错误提示
- 改进导航体验
- 添加过渡动画

## 技术选型

### 测试框架
- JUnit 5 - 单元测试框架
- Mockito - Mock 框架
- SpringBootTest - 集成测试
- H2 - 内存测试数据库

### 代码质量
- ESLint - JavaScript 代码检查
- JSDoc - JavaScript 文档注释
- Lombok - Java 代码简化

### 性能优化
- CSS 动画 - 硬件加速动画
- 懒加载 - 延迟加载非关键资源
- 防抖节流 - 优化事件处理

## 零、最严重的架构问题

### A0. 两套前端代码严重不同步（致命）

**现象：** 项目中存在两套前端代码，内容完全不同：

| 位置 | 用途 | 状态 |
|------|------|------|
| `backend/src/main/resources/static/` | 服务器实际提供的页面 | ❌ 极旧半成品 |
| `frontend/` | 模块化版本 | ⚠️ 较新但未部署 |

**实测证据：**

服务器提供的 `app.js`（`backend/src/main/resources/static/js/app.js`）：
- 174行，缺少 `showLoginPage`、`handleLogin`、`handleRegister`、`logout`、`checkAuth`、`updateAuthUI`、`closeSearch`、`onSearchInput`、`doSearch` 等方法
- `loadEpisodes(dramaId)` 方法体只写了一行 `container.innerHTML = '<p>加载中...</p>';`，没有实际加载逻辑
- `showSearch()` 只有一行 `console.log('显示搜索');`
- `renderDramaDetail` 引用了 `drama.status`（后端已不存在此字段）

**影响：** 用户访问 `http://localhost:8080/` 看到的是旧版半成品页面，几乎所有交互功能都无法使用。

```bash
# 方案2（推荐）：删除 frontend/ 目录，统一使用 backend/src/main/resources/static/
```

## 一、首页问题

### U1. 推荐列表和热播列表内容重复

**现象：** 首页"推荐短剧"和"热播短剧"显示了相同的短剧（如"北派寻宝笔记"同时出现在两个列表中）。

**根因：** `getRecommended` 使用 `findTopRated`（按评分排序），但所有短剧评分都是0，所以结果等同于随机排序。`getHot` 使用 `findByIsHotTrueOrderByViewCountDesc`，但 `isHot=true` 的短剧只有4部，和推荐列表高度重叠。

**修复方案：**

```java
// DramaService.java — 推荐逻辑改为：优先新品 + 随机化
public Page<DramaSummary> getRecommended(int page, int size) {
    return mapToSummaryPage(
        dramaRepository.findByIsNewTrueOrderByCreatedAtDesc(PageRequest.of(page, size))
    );
}
```

或者添加随机推荐接口：

```java
@Query("SELECT d FROM Drama d ORDER BY FUNCTION('RAND')")
Page<Drama> findRandom(Pageable pageable);
```

**Claude Code 执行指令：**

> 修改 `DramaService.getRecommended()`：推荐列表改为按 `isNew=true` + `createdAt DESC` 排序，与热播列表区分开。如果 `isNew` 的短剧不足，fallback 到全部短剧按创建时间倒序。

### U11. 互动点不触发

**现象：** 即使视频能播放，互动点也不会触发弹出。

**根因：** `player.js` 中 `checkInteractionPoints()` 使用 `p.timestamp` 匹配时间，但后端返回的字段名是 `timestampMs`（毫秒），前端比较的是秒。

**修复方案：**

```javascript
// player.js — 修正字段名和单位
checkInteractionPoints() {
    const currentTimeMs = Math.floor(this.currentTime * 1000);
    const point = this.interactionPoints.find(p =>
        Math.abs(p.timestampMs - currentTimeMs) < 1000 && !p.shown
    );
    if (point) {
        point.shown = true;
        this.showInteraction(point);
    }
},

loadEpisode(episode) {
    this.currentEpisode = episode;
    this.interactionPoints = (episode.interactions || []).map(p => ({
        ...p,
        shown: false
    }));
    this.videoElement.src = episode.videoUrl;
    document.getElementById('player-title').textContent = episode.title || '';
},
```

**Claude Code 执行指令：**

> 修改 `player.js`（`backend/src/main/resources/static/js/player.js`）：
> 1. `loadEpisode` 中将 `episode.interactionPoints` 改为 `episode.interactions`，并初始化 `shown: false`
> 2. `checkInteractionPoints` 中将 `p.timestamp` 改为 `p.timestampMs`，将秒级比较改为毫秒级比较
> 3. 确保时间比较容差为1000ms

### U12. 自动连播不工作

**现象：** 视频播放完毕后不会自动播放下一集。

**修复方案：**

```javascript
onVideoEnded() {
    const drama = state.currentDrama;
    const currentEp = state.currentEpisode;
    if (!drama || !currentEp) return;
    const episodes = drama.episodes || [];
    const currentId = currentEp.episodeId || currentEp.id;
    const currentIndex = episodes.findIndex(ep => ep.id === currentId);
    if (currentIndex >= 0 && currentIndex < episodes.length - 1) {
        app.playEpisode(episodes[currentIndex + 1].id);
    }
},
```

**Claude Code 执行指令：**

> 修改 `player.js` 的 `onVideoEnded` 方法：使用 `currentEp.episodeId || currentEp.id` 获取当前剧集ID，然后在 `drama.episodes` 中用 `ep.id` 匹配查找索引，实现自动连播。

```javascript
reportProgress() {

loadEpisode(episode) {
```

```javascript
showResult(result, selectedOptionId) {
    const options = document.querySelectorAll('.interaction-option');
    options.forEach(btn => btn.disabled = true);

    const type = this.currentPoint.type;
    let resultHtml = '';

    if (type === 'QUIZ') {

        resultHtml = this.buildStatsHtml(result);
    } else if (type === 'EGG') {
        resultHtml = '🥚 彩蛋已收集 +5分';
    }

    resultHtml += '<button class="continue-btn" onclick="interaction.close()">继续观看</button>';
    document.getElementById('interaction-result').innerHTML = resultHtml;
},
```

**Claude Code 执行指令：**

> 修改 `interaction.js` 的 `showResult` 方法：
> 1. 根据 `this.currentPoint.type` 区分互动类型（QUIZ/VOTE/CHOICE/EGG）
> 2. QUIZ 类型：从 `this.currentPoint.options` 中找到 `isCorrect` 的选项，与用户选择比较

## 八、安全/配置问题

### U20. preview.html 返回500

**现象：** 访问 `http://localhost:8080/preview.html` 返回 `{"code":500,"message":"服务器内部错误"}`。

**根因：** Spring Security 的 `JwtFilter` 对所有请求（包括静态资源）执行过滤，`preview.html` 请求被当作API请求处理，解析JWT失败后抛出异常。

**修复方案：** 在 `SecurityConfig` 中将 `preview.html` 加入 permitAll 列表，或在 `JwtFilter` 中跳过静态资源。

**Claude Code 执行指令：**

> 1. 在 `SecurityConfig` 中添加 `.requestMatchers("/preview.html").permitAll()`
> 2. 在 `JwtFilter` 中添加静态资源跳过逻辑：如果请求路径以 `.html`、`.css`、`.js`、`.png`、`.jpg` 结尾，直接放行

### U21. JWT 默认密钥仍可使用

**现象：** `application.yml` 和 `application-h2.yml` 中 JWT 密钥默认值为 `a]9kL2mN5pQ8sT1vX3yA6bC0dE7fG4hI`，`JwtUtil.validateSecret()` 未将此值加入黑名单。

**修复方案：** 在 `JwtUtil` 的 `@PostConstruct validateSecret()` 中添加此默认值到禁止列表。

**Claude Code 执行指令：**

> 修改 `JwtUtil.validateSecret()`：添加 `Set<String> forbidden = Set.of("defaultSecretKeyMustBeAtLeast32BytesLong!!", "a]9kL2mN5pQ8sT1vX3yA6bC0dE7fG4hI"); if (forbidden.contains(secret)) throw new IllegalStateException("jwt.secret 不能使用默认值");`

**修复方案：** 添加 `JWT_SECRET: ${JWT_SECRET}` 到 docker-compose.yml。
